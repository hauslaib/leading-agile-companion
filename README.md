# Leading Agile Companion

Working materials for *Leading Agile When No One Agrees*. Scenarios, retrospective formats, and worksheets for the moments the book describes — when teams, leaders, or stakeholders are pulling in different directions and someone has to facilitate a way through.

## What this is

A repository of artifacts used in real engagements: PI planning rooms, leadership offsites, retrospectives that need more than sticky notes. Each file is something I have facilitated or written in service of an actual team. Names and details are anonymised.

## Who it's for

- RTEs, STEs, Scrum Masters, and Agile Coaches looking for facilitation patterns when alignment is the problem.
- Engineering managers running their first cross-team retrospective.
- Readers of the book who want the worksheets pulled out as editable files.

## Contents

- `scenarios/`: Anonymised situations for coaching sessions and SPC-style training. Each scenario gives the setup, the tensions in play, and discussion prompts. No prescriptive answers.
- `retrospectives/`: Formats beyond Start/Stop/Continue, with facilitation notes for groups that disagree.
- `worksheets/`: Conflict maps, stakeholder tension grids, and decision logs as fillable Markdown.
- `discussion-guides/`: Chapter-by-chapter prompts and a six-week book-club structure.
- `examples/`: Filled-in versions of the worksheets so the format is unambiguous.

## How to use it

Clone, copy what you need into your own working folder, edit. Everything is Markdown. The materials stand on their own; chapter references appear where they add context.
